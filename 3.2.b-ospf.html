3.2.b - OSPF

enabled with

router ospf <process-id>
router-id 1.1.1.1
network <address> <wildcard> area <number>

To check which interfaces are participating show ip ospf interfaces brief
To check the database: show ip ospf database

Default timers
...

Hello: 10s on broadcast, 30s on non-broadcast multiaccess (NBMA)
ip ospf hello-interval 10
Dead: 4x the Hello timer
ip ospf dead-interval 40

DR/BDR
...

Who is elected as the Designated Router?

  • First router running OSPF
  • The highest priority on the interface (default 1)
  • The highest router ID
    The Backup Designated Router will be elected following the same rules.
    The other routers will be DROthers
    There is no preempt
    An existing DR cannot be pushed away from its role.

ABR - Area Border Router
...

Router that is in multiple OSPF areas (area 0 and at least 1 other one)
Sends Type 3 LSAs to a zone B for each network (LSA type 2) in zone A.
The routes learned from a type 3 LSA = from different areas, are shown as O - IA (Inter Area) in the routing table

ASBR
...

Autonomous System Boundary Router is connected to a network in a different AS (zone not running OSPF), it redistributes routes from other protocols (EIGRP...)
Generates Type 5 LSAs.
For example:
redistribute static subnets

LSA types
...

  • LSA Type 1:            Router LSA
  • LSA Type 2:            Network LSA
  • LSA Type 3:            Summary LSA
  • LSA Type 4:            Summary ASBR LSA
  • LSA Type 5:            Autonomous system external LSA
  • LSA Type 6:            Multicast OSPF LSA
  • LSA Type 7:            Not-so-stubby area LSA
  • LSA Type 8:            External attribute LSA for BGP

Type 1 - Router
...

show ip ospf database router
Info about a router.
Contains info about the links:

  • amount
  • for each link: to which network it is connected (address and type)
    • stub network = network with NO other neighbor on the segment
    • transit = network WITH other neighbors

Type 2 - Network
...

show ip ospf database network
Describes a network with 2 or more OSPF routers on it.
The DR is responsible of creating and sending this LSA
Sorted by Link ID = IP address of the DR on this link

Type 3 - Network summary
...

show ip ospf database summary
Sorted by Link ID, advertised by an ABR
The ABR is then the next-hop for the route to this network
--> Inter-Area prefix in OSPFv3

Type 4 - ASBR summary
...

show ip ospf database
ABRs have to transmit the location of ASBRs from one area to another
Allows any router in any area to know how to reach an ASBR
--> Inter-Area router in OSPFv3

Type 5 - AS External
...

show ip ospf database external
Routes resulting from those LSAs are shown with "O E2"
--> AS-External in OSPFv3

Type 8 - OSPFv3 Specific
...

Link local information

Adjacencies
...

  • Down: OSPF enabled, but the interface is down
  • Init: Started sending and receiving Hello messages, but the other router doesn't know about me yet
  • 2 - way: Receiving Hellos including me
    Then, on a multiaccess (broadcast or non-broadcast), only if the distant router is a DR or BDR. Or always on a p2p link:
  • Exstart (who starts)
  • Exchange (list the database index = Database Descriptor (DBD), only keys no values)
  • Loading (Synchronization of the databases via LS requests and LS updates)
  • FULL (databases synchronized)
    debug ip ospf adjacencies

Hello packets
...

OSPF is IP protocol 89

  • Sent to the multicast 224.0.0.5 on broadcast link types
  • Sent via unicast on non-broadcast links, the neighbor IP address needs to be manually specified
    To configure OSPF on a non-broadcast link:
    router ospf <pid> neighbor <neighbor_ip>
    An interface can be forced to the non-broadcast type with ip ospf network non-broadcast

Cost
...

Calculated using
By default, the reference bandwidth is 100, so any link > 100Mbps has a cost of 1
To change it to 10G: auto-cost reference-bandwidth 10000
You can also change the cost on a specific interface using ip ospf cost <cost>

Summarization
...

ABRs can summarize type 3 LSAs to reduce the amount of routes advertised to other areas.
Network numbers in areas should be assigned contiguously.
``
ASBR can summarize type 5 LSA.
To manually summarize:

  • Between areas, on the ABR: area <id> range <subnet> <mask>
  • Between AS, on the ASBR: summary-address <subnet> <mask>
    in both cases, the mask is NOT a wildcard mask

Route filtering
...

You can filter some routes locally on your router, so they won't be used to forward traffic. They will remain in the OSPF database.
Distribute Lists allow you to control the routing advertisements sent, received and redistributed by your router. They use access lists + prefix lists to match the routes concerned

router ospf 1
distribute-list <prefix|route-map|ACLname|> <in|out>

OSPFv3
...

Unlike OSPFv2, does NOT use MD5 but IPSec

Uses IPv6 Link-local addresses to exchange routing information. The next-hop addresses in the routing table will be link-local addresses as well.
Destination addresses are FF02::05 and FF02::06

OSPFv3 modified the contents of the router LSA to remove the network addressing. This information is now advertised in the intra-area prefix LSA and the link-local LSA.

Supports both IPv6 and IPv4.
You need to explicitly enable the interface to participate in IPv4 routing by using the ospfv3 process-id ipv4 area-id command.
A reserved instance ID in the range 64 to 95 identifies a neighbor adjacency as being used for IPv4.
Instance ID allow to support Address Families (AF):

  Instance ID 0-31     IPv6 unicast AF
  Instance ID 32-63     IPv6 multicast AF
  Instance ID 64-95     IPv4 unicast AF
  Instance ID 96-127    IPv4 multicast AF
  Instance ID 128-255    Unassigned
