1.4 - SD-WAN

Traditional WAN challenges
...

Interconnect a lot of modules (soho, branches, HQ, DC, BDC)
Often redundant links, with backup Service provider.
Different types of links (4G/5G, VPNs, fiber, MetroE)
MPLS provides way more reliability compared to "the Internet"

Advantages of SD-WAN
...

Tunnels are really useful, their only downside is the configuration complexity. Using SD-WAN simplifies this complexity by creating tunnels automatically
Cisco SD-WAN uses IPSec tunnels.
only edge routers need to smart.
A central controller allows to define app based policies that will be pushed on the edge routers.

Cisco SD-WAN
...

originally made by Viptela, acquired by Cisco. Meraki also has its own, different SD-WAN solution
4 types of components:

  • vManage
    Is the management plane where you define the configurations/policies using a GUI
  • vSmart
    Is deploying this control plane
  • vEdge
    Edge router, can be a cEdge (ISR 4K running Viptela code, )
  • vBond
    out-of-band orchestrator, used by vEdges to sync, and for NAT traversal
    the only component that needs a public IP address, as it is used by vEdge and vSmart to establish DTLS tunnels

Management
...

One dashboard, some sandboxes are available online
Uses templates for configuration.
vAnalytics can be purchased separately

Topologies
...

How do we setup our IPSec tunnels between our vEdges?
Depends on the licensing. The cheapest is a hub and spoke.
Can be upgraded to a partial, full mesh, or even point-to-point.
When you have 2 different WAN circuits, you can loadbalance the traffic:

  • active/active, can be weighted
  • active/standby pinning depending on the application
  • use application-aware SLA: when jitter rises on a certain application with SP A, let's switch to SP B. How to detect the application?
    • Licensed: DPI (Deep-Packet Inspection)
    • free: 6 tuple (Source/dest IP, source/dest port, QoS marking and IP protocol)

Programmability
...

SD-WAN's vManage exposes Netconf and REST APIs
vManage actually uses Netconf to push configs to vSmart, which then uses OMP (Overlay Management Protocol) to push config to vEdges.
