3.2.c - eBGP

AS numbers between 1 and 65535
1-64511 assigned by the IANA
64512-65535 reserved for documentation/private use

32-bits AS numbers have been introduced to extend the AS numbers pool

TCP is designed to be scalable, reliable and stable

  • Reliability
    1. Periodic TCP keepalives
    2. Incremental, triggered updates
  • Scalability
    1. 800K+ internet routes
    2. TCP sliding window ACKs
  • Stability
    1. Batched updates
    2. Slower convergence

Internal BGP routes AD: 200
External BGP routes AD: 20

BGP uses 4 types of messages:

  • open: first message to initiate the neighborship
  • Keepalive: periodically sent to maintain the relationship
  • update: containing the full routing table or a partial update
  • notification: sent before closing the BGP connection. Contains an error code, subcode and data.

BGP neighbors
...

BGP Speaker is a router that runs BGP.
BGP peer is a speaker configured to form a neighbor relationship.
The neighbor can be inside the same AS (IBGP) or outside (EBGP).
IBGP is often used for redundancy and load-balancing

For two routers to be EBGP neighbors, they need to:

  • be defined as neighbors, with an established TCP session
  • directly connected and use IP addresses reachable from each AS
  • have different AS numbers

Path selection
...

Routing policy is based on the following attributes:

  • Prefer the highest weight attribute (local to router)
  • Prefer the highest local preference attribute (global with AS)
  • Prefer route originated by the local router (next hop = 0.0.0.0)
  • Prefer the shortest AS path (least number of autonomous systems in AS-path attribute)
  • Prefer the lowest origin attribute (IGP < EGP < incomplete)
  • Prefer the lowest MED attribute (exchanged between autonomous systems)
  • Prefer an EBGP path over an IBGP path
  • (IBGP route) Prefer the path through the closest IGP neighbor (best IGP metric)
  • (EBGP route) Prefer the oldest EBGP path (neighbor with the longest uptime)
  • Prefer the path with the lowest neighbor BGP router ID
  • Prefer the path with the lowest neighbor IP address (multiple paths to the same neighbor)

When faced with multiple routes to the same destination, BGP chooses the best route for routing traffic toward the destination by following the route selection process.

When a router has multiple AS exit points, it can use different weights to influence the path selection process.
Routes with a higher weight are preferred

Path Attributes
...

Specified for each route in a BGP update

Transitive attributes should be propagated even if not understood by the local router

Local preference is only exchanged via IBGP

The Multi-Exit Discriminator (MED) is initially exchanged between autonomous systems.
It is set to influence inbound traffic on multiple entry points from another AS.
A lower MED value is preferred
