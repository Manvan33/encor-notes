Queuing mechanisms
...

Queuing happens when we have congestion in our network.
There are many different queuing mechanisms. Older methods are insufficient for modern rich-media networks. However, you need to understand these older methods to comprehend the newer methods:

  • FIFO is a single queue with packets that are sent in the exact order that they arrived.
    • No QoS involved
  • PQ (Priority Queuing) is a set of four queues that are served in strict-priority order. By enforcing strict priority, the lower-priority queues are served only when the higher-priority queues are empty.
    • This method can starve traffic in the lower-priority queues.
  • CQ (Class Queuing) is a set of 16 queues with a round-robin scheduler. To prevent traffic starvation, it provides traffic with bandwidth guarantees.
    • The drawback of this method is that it does not provide strict priority for real-time traffic.
  • WFQ divides the interface bandwidth by the number of flows, ensuring proper distribution of the bandwidth for all applications.
    - Lower bandwidth flows get more priority
    - Provides a good service for the real-time traffic, but there are no guarantees for a particular flow.
    Cisco's mechanism:
    Priority Queuing-Class Based Weighted-Fair Queuing - PQCBWF = Low Latency Queuing - LLQ

Congestion management and Avoidance
...

We want to avoid Tail Drop (when queues get filled up)
Weighted Random Early Detection (WRED) prevents all the TCP congestion windows to sync by dropping random packets in the queue, while choosing which classes are more susceptible to be dropped.

Define an Overall QoS Policy
...

The MQC structure allows you to define a traffic class, create a traffic policy, and attach the traffic policy to an interface.

  1. Class map

    • Define classes of traffic.
    • "Which traffic do we care about?"

  2. Policy map

    • Define QoS policies for classes.
    • "What will be done to this traffic?"

  3. Service map

    • Apply a service policy.
    • "Where will this policy be implemented?"

Policy-Based Routing (PBR)
...

This feature can be used to optimize the traffic path for a specific application
Policy-based routing is a process whereby the device puts packets through a route map before routing them. The route map determines which packets are routed to which device next. You might enable policy-based routing if you want certain packets to be routed some way other than the obvious shortest path.
