Overview

Need for STP
...

We want to eliminate

  • Broadcast storms: Each switch on a redundant network floods broadcasts frames endlessly. Switches flood broadcast frames to all ports except the port on which the frame was received. These frames then travel around the loop in all directions.
  • Multiple frame transmission: Multiple copies of the same unicast frames may be delivered to a destination station, which can cause problems with the receiving protocol. Many protocols expect to receive only a single copy of each transmission. Multiple copies of the same frame can cause unrecoverable errors.
  • MAC database instability: This problem results from copies of the same frame being received on different ports of the switch. The MAC address table maps the source MAC address on a received packet to the interface it was received on. If a loop occurs, then the same source MAC address could be seen on multiple interfaces, causing instability. Data forwarding can be impaired when the switch consumes the resources that are coping with instability in the MAC address table.

Process
...

STP election starts by electing a root switch.
Then on each switch:

  • Selection of one root port, closer to the root switch
    • All other ports are then designated ports, used to send data to non-root switches
  • Some designated blocked ports are disabled by STP to eliminate loops-> become non-designated ports

Operation
...

BPDU (Bridge Protocol Data Units)
...

2 types:

  • Configuration, sent by the root switch only
  • Topology Change Notification (TCN), sent to the root switch
    • The root switch then answers with a Topology Change Acknowledgment (TCA), which is a TCN, but with the ack bit set to 1.

When the root switch receives a TCN BPDU, it generates a new configuration BPDU with the topology change flag set. When other switches receive this BPDU, they set the MAC address aging timer equal to the forwarding delay timer, which is 15 seconds. Entries that have not communicated during the past 15 seconds are aged out of the MAC address table.

BPDUs are sent to the destination mac address 01:80:c2:00:00:00

Root bridge
...

Lowest Bridge ID (BID) = Bridge priority + Mac Address
Bridge priority: 0-65535, default 32768 + VLAN number

root ports
...

Port which has the lowest cost to the root switch
Cost of the path is based on the speed of the link. 1Gb/s = 4
In case of a tie: Root switch sends (in BPDUs) a different port ID for each interface, the lowest port ID is chosen

Designated ports
...

On each segment (link) not linked to root bridge, the switch with the lowest BID is designated, its interface is designated, the other one is blocked.

Spanning-Tree Standards
...

Standard Features
IEEE 802.1D Legacy standard for bridging and STP CST(Common Spanning Tree): Assumes one spanning-tree instance for the entire bridged network, regardless of the number of VLANs
PVST+ A Cisco enhancement of STP that provides a separate 802.1D spanning-tree instance for each VLAN that is configured in the network
802.1s (MSTP) Maps multiple VLANs into the same spanning-tree instance
802.1w (RSTP) Improves convergence over 1998 STP by adding roles to ports and enhancing BPDU exchanges
Rapid PVST+ A Cisco enhancement of RSTP using PVST+

Multiple Spanning Tree Protocol
...

The main purpose of MST is to reduce the total number of spanning-tree instances to match the physical topology of the network. Reducing the total number of spanning-tree instances will reduce the CPU loading of a switch. The number of instances of the spanning tree is reduced to the number of links (that is, active paths) that are available.

Rapid Spanning Tree Protocol (RSTP)
...

RSTP categorizes links into different types to optimize the spanning tree topology.

  • Point-to-Point link: direct connections between network devices, and full-duplex communication is assumed.
    This allows RSTP to converge more quickly and efficiently in such situations.

PortFast and BPDU Guard
...

PortFast allows to skip the delay caused by STP on interfaces connected to end-devices.
BPDU Guard is meant to be used on PortFast-enabled interfaces, to prevent the creation of loops if a switch is plugged in an access port. If a BDPU is received, the interface is shut down.

The spanning-tree bpduguard enable interface configuration command configures BPDU guard on an interface. The spanning-tree portfast bpduguard default global configuration command enables BPDU guard globally for all PortFast-enabled ports.

Loop Guard
...

Some links are prone to unidirectional link failure (i.e. fiber) that would cause STP loops.
One of the methods that can solve this is Loop Guard. When a switch is sending but not receiving BPDUs on the interface, Loop Guard will place the interface in the loop-inconsistent state and block all traffic.
This can be configured on root or alternate ports.

Unidirectional Link Detection (UDLD) is a Cisco proprietary layer 2 protocol used to determine the physical status of a link.

Root Guard prevents a configured port from becoming a root port. Root guard prevents a downstream switch (often misconfigured or rogue) from becoming a root bridge in a topology.
