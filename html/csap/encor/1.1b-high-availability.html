HSRP - Hot Standby Router Protocol
...

Creates a phantom router with the gateway @IP, that can be configured on multiple routers.
An election happens, creating an Active and a Standby router. The highest priority wins (or the highest IPv4).
Default priority is 100

  • Every 3 seconds, standby and active routers exchange Hello packets
    When no hello has been heard from the active for more than the "whole" time (10s by default), the standby router takes over.
    States:
  1. Init
  2. Learn
  3. Listen
  4. Speak
  5. Standby
  6. Active

HSRP Specifics
...

  • Cisco Proprietary, RFC 2281
  • Active and "Standby" routers
  • Version 1: VR @MAC = 0000.0c07.acXX
    • XX= group id
  • @IP Multicast for Hello = 224.0.0.2
  • Version 2: VR @MAC = 0000.0c9f.fXXX
  • @IP Multicast for Hello = 224.0.0.102
  • ms-precision on hello timers, whole timer > 3xhello timer
    --> <1s failover delay (only version 2)
  • Standby keeps the active state even when the failed router comes back.
    --> Preemption has to be enabled for the 1st router to become active
  • Cannot use a real interface @IP for the virtual router

Config
...

R1(config)#int gig 0/1
R1(config-if)#standby 10 ip 10.1.1.1
R1(config-if)#standby 10 preempt
R1(config-if)#standby 10 priority 110
R1(config-if)#end

R2(config)#int gig 0/1
R2(config-if)#standby 10 ip 10.1.1.1
R2(config-if)#standby 10 preempt
R2(config-if)#end

VRRP - Virtual Router Redundancy Protocol
...

Specifics
...

  • Open Standard, RFC 3758
  • Master and Backup Routers
  • VR @MAC = 0000.5E00.01XX
  • Preempt enabled by default, default priority of 100
  • VR @IP can be used on a physical interface too
  • Multicast @IP = 224.0.0.18
  • Master Advertisement Interval (1s),
  • Master Down Interval = 3* MAI + [(256-VRRP Priority)/256]
  • default priority is 100

Configure
...

interface vlan 1
ip address <gw-ip> <mask> secondary
vrrp <group> priority <priority>
vrrp <group> authentication text <key-string>
vrrp <group> authentication md5 key-string <key-string>

Gateway Load Balancing Protocol (GLBP)
...

Cisco Proprietary
Load balancing, not only failover
Up to 4 routers:

  • 1 Active Virtual Gateway (AVG) respond to ARP queries for @MAC of the default gateway. It can be a AVF too
  • Up to 4 Active Virtual Forwarders (AVF)
  • AVG answers different @MAC to different queries, to balance the load following different algorithms :
    • Round-Robin
    • Host-dependent
    • Weighted

Layer 2 redundancy
...

Redundant supervisor modules can be configured in several modes.

Redundancy ModeBehavior When Active Module FailsFailover Time
RPR - Route Processor RedundancyStandby module reloads every other module, initializes all supervisor functions.>2 minutes
RPR+Standby module finishes initializing without reloading other modules.>30 seconds
SSOStandby module is already initialized.>1 second

You can enable another redundancy feature along with SSO. Cisco Nonstop Forwarding (NSF) is an interactive method that focuses on quickly rebuilding the Routing Information Base (RIB) table after a supervisor switchover.
