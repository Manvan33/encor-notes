Frame types
...

  • Data frames: Clear-text header containing MAC Address, (encrypted) data, CRC
    • Null Frame: Sent by a client going into power saving mode to inform the AP
  • Management frames: Beacon and probe frames, authentication and association frames, de-authentication and re-association
  • Control frames: Request to Send, Clear to send, Acknowledgment

Discovering wireless
...

Beacons frames, sent by APs, used by clients for passive scanning, contain:
SSID, Channel, rules (incl data rates, encryption, power saving)
Probe frames, sent by clients during active scanning, contain:
SSID, client capabilities
Probe responses sent by APs

Joining a wireless AP
...

The client sends an Auth Request
AP sends an Auth response, containing a challenge to solve.
Once authorized:
The client sends an association request, and then receives an association response.

To leave:

  • De-authentication, can be sent by either the client or the AP
  • Disassociation, sent by the client. The client auth info is then saved. It allows quick roaming between access points, without the need to re-authenticate

Wireless flow control
...

Every client will have an IFS (Inter-Frame Spacing) timer, increased by the duration of the frames from other clients that he senses.
RTS and CTS are used to allow compatibility between different 802.11 standards. An 802.11n device may not be able to sense the carrier for 802.11ac frames.

Split-MAC
...

Splitting of 802.11 functions between:

  • the AP, which processes real-time portions of the protocol,
  • and Cisco WLC, which manages items that are not time-sensitive

CAPWAP
...

Control and Provisioning of Wireless Access Points (CAPWAP) is an open protocol that enables a WLC to manage a collection of wireless APs
2 tunnels are built:

  • 1 DTLS (encrypted) for control plane, on port UDP 5246
  • 1 unencrypted by default for data plane, on port UDP 5247

WLC functions: MA & MC
...

Both can run on the same WLC, or on separate ones.

Mobility Agent (MA) terminates CAPWAP tunnels, maintains the client database. MA also configures and enforces security and QoS policies for wireless clients.

Mobility Controller (MC) provides roaming management, Radio Resource Management (RRM), wireless intrusion prevention, and guest access. Functions provided on a system-wide basis.
MA reports local and roamed client states to MC. The MC builds a database of client stations across all the MA.

Roaming
...

BSS = SSID served by only one AP (BSSID)
ESS = Service Set (same SSID) served by multiple APs

Autonomous roaming
...

When the RSSI of AP2 becomes stronger than AP1, the client will authenticate with AP2, and then send a Re-Association request to AP2. If successful, the client will roam to AP2, seamlessly.

Controller-led L2 roaming
...

The client is using RSSI values to find the best AP candidate to roam to.
The WLC is going to update the endpoint location from AP 1 to AP 2. The endpoint will be able to keep its IP address. It's a layer 2 roaming.

Multiple WLCs, part of a mobility group.
When the endpoint sends an auth request to AP @, WLC2 will ask WLC1 for the client information, and keep it in its own database. The client will roam seamlessly.

Layer 3 roaming
...

If AP 2 is in a different WLC (but part of a mobility group with WLC 1), in a different VLAN and Subnet, roaming will probably break a lot of IP connections.
When the client asks for roaming to AP 2, WLC 2 is going to ask WLC 1 for the client's IP address. When this address is in a different subnet than the data subnet for WLC2, WLC1 will become an Anchor WLC for the client. All the traffic from this client will then be sent over an inter-WLC CAPWAP tunnel to WLC1.
This can be used to send traffic from a Guest SSID to a WLC in a DMZ, behind a firewall forbidding access to the internal network.

WLC roles
...

PoP = Point of Presence (WLC containing the IP - MAC@)
PoA = Point of Attachment (WLC attached to the client's AP)

Local: The controller provide both PoP & PoA
Anchor: The controller provide PoP only & always paired with a foreign controller (PoA)
Foreign: The controller provide PoA only & always paired with a Anchor controller (PoP)
Export Anchor: The controller provide PoP only and always paired with an export foreign controller.
Export Foreign: The controller provide PoA only and always paired with an export anchor controller.

AP-WLC Discovery
...

When the LAP joins the network, it gets an IP address, default gateway and DNS via DHCP.
It will then try to discover a WLC via:

  • DHCP option 43 containing the controller address
  • DNS: query A record for cisco-capwap-controller.<mylocaldomain.lan>
  • Cache: using the last known WLC address
  • Broadcast: 255.255.255.255 to find a WLC on the same subnet
  • Static IP configured in the console
  • Mobility group cache: using the IP addresses of all WLC in the same mobility group as the last known WLC

The LAP stores all the discovered WLCs in a list and then:

  • It will try to join them following the order that was configured on the LAP (primary, secondary, etc.)
  • If no priority was configured, it will try to join all of them.
    • If one WLC claims to be a Master WLC, it will join this one, and get a priority list of WLCs.
    • If no master WLC: joins the WLC with the fewest connected APs

Once its connected, it gets the software image to use, the configuration to apply with all the SSIDs and auth methods, etc.

Adaptive Wireless Path Protocol (AWPP)
...

Mesh Access Points use AWPP to determine the best path through the other mesh access points to the controller (Root Access Point). It uses a dedicated radio for backhaul operations.

OfficeExtend Access Points (OEAP)
...

A Cisco OEAP provides secure communications from a WLC to a Cisco AP at a remote location, seamlessly extending the corporate WLAN over the Internet to an employee’s residence.
The user’s experience at the home office is exactly the same as it would be at the corporate office.
Datagram Transport Layer Security (DTLS) encryption between the access point and the controller ensures that all communications have the highest level of security.

Authentication
...

WPA2 is 802.11i -compliant and is the current standard for enterprise networks

  • WPA2 Personal mode uses a PSK for authentication
  • WAP2 Enterprise mode uses EAP authentication, but necessitates a RADIUS server

Some devices, such as printers or cameras, can use MAC Authentication Bypass (MAB). MAB enables port-based access control using the MAC address of the endpoint.

WPA3
...

WPA3 several improvements: stronger encryption, robust authentication methods, and protection against common security vulnerabilities.
i.e., Simultaneous Authentication of Equals (SAE), also known as Dragonfly. SAE replaces the pre-shared key (PSK) authentication used in WPA2 with a more secure key exchange mechanism.

PSK
...


The PSK is used for authentication and traffic encryption

Open authentication
...

  • operates at Layer 1 and 2
  • Used by default by 802.11 compliant devices
  • only "authenticates" a device as 802.11 compliant

802.1X
...

The 802.1X protocol defines port-based access control. The protocol defines three roles:

  • Supplicant: The client that wants to access the network.
  • Authenticator: The point of access (a switch, an AP, or a WLC). The authenticator is the point of entrance to the network.
  • Authentication server: A machine, somewhere in the network, that keeps a list of conditions by which access is granted or refused.

Each supplicant is authenticated individually.
While authenticating each other through 802.1X, the client and RADIUS servers derive an individual session key.
This key will be used to encrypt traffic.
