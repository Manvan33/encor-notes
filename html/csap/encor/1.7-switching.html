CAM table - Content Addressable Memory
...

Stores the MAC-Port-VLAN table
When forwarding:

  • Stores the source MAC address if it is not already in the CAM table
  • Destination MAC address Look up in the CAM table
  • Unknown unicast flooding when the destination MAC isn't found in the CAM table.
  • Aging timers empty the CAM table regularly. The default aging-time for CAM entries is 300 seconds

the default aging time for ARP table entries is 4 hours.

TCAM - Should you try to forward the frame?
...

Memory for ACLs and QoS.
Cisco switches look up in both the CAM and TCAM at the same time --> no performance loss when using ACLs

L3 Switching mechanisms
...


The fast-switching cache is only used for packets in one flow, and built at the beginning of this flow


The CEF cache, also known as FIB (Forward Information Base) is built directly, and used for multiple flows.
The adjacency table contains the L2 headers rewrite information.

Exceptions
...

Not all packets can be processed in the hardware. When traffic cannot be processed in the hardware, it must be received by software processing of the Layer 3 engine. This traffic does not receive the benefit of expedited hardware-based forwarding.
Exemples:

  • They use IP header options.
  • They have an expiring IP TTL counter.
  • They are forwarded to a tunnel interface.
  • They arrive with unsupported encapsulation types.
  • They are routed to an interface with unsupported encapsulation types.
  • They exceed the Maximum Transmission Unit (MTU) of an output interface and must be fragmented.

Control and Data plane
...

In L3 devices,  the exchange of routing information is performed in the control plane, and data packets are forwarded by the data plane.
