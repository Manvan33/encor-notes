VXLAN
...

In big L2 Networks i.e. datacenters, 4096 VLANs is not enough, and STP doesn't scale.
We need layer 2 connectivity to allow for live migrations between servers.
We need more VLANs, deployed dynamically on different leafs switches.
The first solution was to use PVLANs but that was causing too much overhead.

We want to have the scalability of a Layer 3 domain with the simplicity of L2 communication.

The entire L2 frame is encapsulated in a VXLAN header, then in a UDP header (port 4789), then IP header. = MAC in IP / UDP encapsulation.
The header is now larger by
20+8+8+14 = 50 bytes overhead (or 54 if the VLAN tag is in the internal eth header).
IP+U+V+Eth

The VXLAN header:
...

VNID = VxLAN Network Identifier, over 24 bits = 16 millions different IDs
The tunnel entry point is called a VTEP
VXLAN is different from LISP because it's data plane only. To make it work, you should pair it up with a control plane protocol. It can be MP-BGP, COOP (ACI) or even LISP (SDA)
