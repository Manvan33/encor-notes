1.1 - Enterprise Network Architecture

The modules
...

  • Entreprise Campus
  • Entreprise Edge
  • Service Provider Edge
  • Remote Location

The goal it to create a reliable and versatile network design. This network should be:

  • Self-healing: Continuously on and available.
  • Self-defending: Protecting the organization and its users.
  • Self-optimizing: Adapting to changing needs beyond the limits of basic standards.
  • Self-aware: Driving change through insight into network activity.

The model
...

Campus entreprise model defined by 3 layers:

  • Core
  • Distribution
  • Access

It can be either Switched or Routed, depending on the layer of the Distribution-Access link.

  • Switched: VLANs terminate at the Distribution layer. Easier and more cost-effective, but STP blocks half of the uplinks.
  • Routed: VLANs terminate at the Access level uplink ifaces. The uplink towards Distribution uses IP routing and has more bandwith

Access
...

Made of switches. No redundancy, each end-device linked to only one access switch
Access switches provide many services to end-devices:

  • VLAN assignment
  • QoS Tagging, including defining the trust boundary (the device acting as the trust boundary decides whether or not to trust the QoS/VLAN information from the end-device)
  • Security: 802.1x authentication, and port-security (limiting the allowed mac addresses and/or their amount)
    Provide resiliency, but no redundancy:
  • Priority to certain devices
  • Minimise interference
  • Visibility on traffic using [NBAR](Network Based Application Recognition): layer 4 to 7 recognition of traffic type to optimize its network resources usage.
